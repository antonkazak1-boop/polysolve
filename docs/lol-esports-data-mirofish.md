# LoL esports → Polysolve → MiroFish

Цель: тянуть **факты по про-LoL** (матчи, форма команд, драфты, KDA и т.д.), нормализовать в текст/JSON и скармливать **MiroFish** как «семена» (по аналогии с [`mirofish-test-seed.md`](./mirofish-test-seed.md)).

---

## 1. Riot Games (официальный API) — когда подходит

- **Портал:** [developer.riotgames.com](https://developer.riotgames.com/) — ключ, лимиты, правила продукта.
- **Сильные стороны:** `match-v5` (детали матча, timeline), аккаунты по Riot ID, ранкед-лестницы, **если уже есть `matchId` / PUUID**.
- **Слабое место для Polymarket:** **нет** готового публичного API уровня «все матчи LEC/LCK, турнирная сетка, агрегаты как на аналитических сайтах». Про-матчи часто нужно **сначала** обнаружить через esports-источник (см. ниже).

**Вывод:** Riot — **истина по одному матчу** (после получения ID), но **не замена** [gol.gg](https://gol.gg/esports/home/) для «домашней страницы лиг и сводной статистики» из коробки.

---

## 2. [Games of Legends (gol.gg)](https://gol.gg/esports/home/)

- **Плюсы:** удобные агрегаты под беттинг/анализ, лиги (LCK, LPL, LEC, …), команды, игроки, чемпионы.
- **Минусы для продукта:**
  - **Парсинг HTML** ломается при любом редизайне.
  - **ToS / robots.txt** — нужно прочитать и не нарушать; для коммерции часто безопаснее **не скрейпить**, а договориться или использовать легальный feed.
  - **Premium** — часть данных может быть за Patreon; не рассчитывать на стабильный бесплатный парсинг «всего».

**Практика:** если идёте в gol.gg — изолировать в отдельный `adapter`, кэшировать, лимитировать частоту, иметь fallback (ручной JSON / другой API).

---

## 3. Платные / специализированные esports API

Для **стабильного** продакшена под рынки вида «кто выиграет серию / турнир» часто берут **Esports API** (PandaScore, и т.п.) — матчи, составы, live, история. Это **замена боли скрейпа** gol.gg, но с биллингом.

Имеет смысл сравнить цену/покрытие LoL с вашими конкретными рынками на Polymarket.

---

## 4. Формат «семян» для MiroFish (совместимо с текущим шаблоном)

Расширяйте блок **«Данные (вставить целиком)»** из [`mirofish-test-seed.md`](./mirofish-test-seed.md):

### 4.1 Мета

- Рынок Polymarket (вопрос / condition slug / дедлайн).
- **Снимок данных:** `as_of` (UTC), **источник** (`riot_match_v5` | `gol_gg_snapshot` | `manual`).

### 4.2 Таймлайн (факты, не интерпретация)

Таблица или список: дата → событие (патч, результат матча, замена в составе, травма если есть в источнике).

### 4.3 Сырой JSON (для модели)

Один компактный объект, например:

```json
{
  "source": "riot_match_v5",
  "region": "EUROPE",
  "match_id": "EUW1_1234567890",
  "teams": [{"name": "G2", "win": true}],
  "patch": "15.5",
  "duration_sec": 1987,
  "stats_summary": "..."
}
```

Для gol.gg — аналогично, но поля из вашего парсера (`league`, `bo`, `side`, `picks_bans`, KDA-агрегаты **как в выгрузке**, без выдумывания).

### 4.4 Ограничения (обязательно)

- Что **не** покрыто источником (например, только последние N матчей, нет live).
- Задержка данных vs время резолюции рынка.

## 5. Пайплайн в Polysolve (предлагаемые слои)

| Слой | Роль |
|------|------|
| **Ingest** | HTTP-клиент: Riot (match by id) и/или gol.gg adapter / внешний esports API |
| **Normalize** | Единая схема `LoLEsportsSnapshot` (команды, результат, время, лига, сырой ref) |
| **Cache** | SQLite/Redis/file по `match_id` + TTL, чтобы не упереться в rate limit |
| **Export** | Генерация markdown-блока или JSON для вставки в MiroFish + опционально привязка к `marketId` в вашей БД |

Крон: раз в X минут для **списка отслеживаемых** матчей/серий (из ваших алертов или ручного конфига).

---

## 6. Что сделать в репозитории дальше (короткий бэклог)

1. **Env:** `RIOT_API_KEY`, опционально `ESPORTS_API_KEY` (если возьмёте провайдера).
2. **Конфиг:** YAML/JSON «какие лиги / команды / матчи» связаны с какими рынками Polymarket (у вас может не быть автоматического маппинга — начните с ручного).
3. **Скрипт CLI:** `npm run lol:snapshot -- --match EUW1_xxx` → печать markdown для MiroFish.
4. **Позже:** эндпоинт `GET /api/esports/lol/snapshot?...` для UI.

Минимальный каркас Riot уже в репо: `backend/src/integrations/riot-lol/riot-lol-client.ts`, CLI: `npm run lol:riot-match -- <matchId>` (нужен `RIOT_API_KEY` в `.env`).

---

## 7. Резюме выбора источника

| Нужно | Лучше начать с |
|--------|----------------|
| Один конкретный матч, детали, timeline | **Riot** `match-v5` при известном `matchId` |
| Сводка по лиге / долгая история как на gol.gg | **gol.gg-подобный скрейп** (риски) или **платный esports API** |
| Быстрый прототип под MiroFish | Ручной копипаст + JSON + затем Riot для 1–2 матчей |

Если скажешь, **какие именно рынки Polymarket** (примеры вопросов), можно сузить схему полей под один сценарий и набросать первый `LoLEsportsSnapshot` + пример готового блока для MiroFish.
