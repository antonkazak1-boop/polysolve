# LoL на Polymarket: какие рынки, какие данные, edge и Monte Carlo

Опрос API: `GET https://gamma-api.polymarket.com/events?tag_slug=league-of-legends&active=true&closed=false` (число событий плавает; на момент проверки было **~30** активных event’ов).

---

## 1. Какие типы рынков есть (по факту с Polymarket)

| Кластер | Примеры с Polymarket | Что нужно из данных |
|--------|----------------------|---------------------|
| **Сезон / плейоффы лиг** | «LoL: LCK 2026 Season Winner», «LoL: LPL 2026 Season Winner» — победитель из списка команд | Регулярка + плейофф: форма, карта силы, head-to-head, патч; часто **нет одного matchId** — модель на турнирную сетку |
| **Международка (регион)** | «LoL: Worlds 2026 Winning Region», «LoL: MSI 2026 Winning Region» | Агрегированная сила регионов, кто квалифицировался, история межрегионалок |
| **Квалы / спец** | «Which teams will qualify to MSI 2026?», Golden Road Gen.G, Faker Finals MVP | Календарь матчей, составы, иногда чисто нарративные рынки |
| **Матч BO3/BO5** | «LoL: GIANTX vs Fnatic (BO3)», «LYON vs JD Gaming (BO5)» | Исход серии, иногда **победитель карты** (Game 1 Winner…) |
| **Гандикап / тоталы** | Game Handicap -1.5/+1.5, Games Total O/U 2.5 | Распределение длины серии и карт; для -1.5 нужна модель **по картам**, не только «кто выиграл серию» |
| **Пропы (микро)** | Total Kills O/U по игре, First Blood, драконы, барон, квадра | **Live** или post-game детализация (timeline, события); иначе только слабые эвристики |

Отдельно в том же теге попадают **не чисто киберспортивные** рынки (стримеры, мемы) — их отсекать по `slug` / ключевым словам, если цель только про-LoL.

## 2. Live-данные + сравнение с рынком (edge)

**Идея:** на каждый рынок хранишь **implied probability** с CLOB (`p_mkt`) и **оценку модели** (`p_model`).

- **Edge (простой):** `edge = p_model - p_mkt` (или в лог-оддс для симметрии).
- **Условный edge (live):** во время серии подставляешь состояние *0–1*, счёт по картам, драфт если есть → обновляешь `p_model`, снова сравниваешь с **текущей** ценой.

**Откуда live:**

- Дешёво: **опрос цены Polymarket** (у вас уже есть инфраструктура) + **Riot match-v5 / timeline** если есть `matchId`.
- Надёжно для про-матчей: **платный esports API** с WebSocket (см. `lol-esports-data-mirofish.md`) — иначе задержка и дыры по матчам.

**Минимальный пайплайн в Polysolve:**

1. Список `conditionId` / `tokenId` для LoL-event’ов (из Gamma или ваша БД).
2. Поток **mid / best bid-ask** с CLOB.
3. Поток **состояния матча** (score, игра N, по возможности драфт).
4. Сервис `edgeCalculator`: на вход `{ market_type, state, p_mkt }` → на выход `{ p_model, edge, confidence }`.

## 3. Monte Carlo: куда логично применять

Monte Carlo имеет смысл, когда можно **симулировать много исходов** из простых случайных величин с оценёнными параметрами.

### 3.1 Серия BO3 / BO5 (победитель матча)

- Задаёшь **вероятность победы сильнейшей стороны на одной карте** `p_map` (из elo/формы/рынка-дерива).
- Учитываешь **синюю сторону / порядок карт** (опционально два `p_map`).
- Симуляция: серия до 2 или до 3 побед на карте → частота «Team A выиграл серию» = оценка `p_model` для YES.

### 3.2 Games total O/U 2.5 (BO3)

- Исход 2–0 или 2–1: `P(2-0) = p² + (1-p)²` при симметрии не совсем так — правильнее два independent или марков цепь по картам.
- MC: тысячи серий → доля серий из **3 карт** vs **2 карт** → вероятность Over 2.5.

### 3.3 Гандикап -1.5

- Для «фаворит -1.5» нужен **3–0 или 3–1** (BO5) или **2–0** (BO3) — явно из симуляции карт.

### 3.4 Kills O/U (сложнее)

Нужна модель **киллов за карту** (распределение Пуассона/негативное биномиальное + корреляция команд). Параметры калибруются по истории (PandaScore post-match и т.д.). MC: симулировать карты → суммарные киллы → доля Over.

**Пропы «first blood / дракон»** — ещё тоньше (ранние фазы); без live-событий или богатой истории MC часто **шум**, edge иллюзорный.

## 4. Практический порядок работ

1. **Инвентаризация:** скрипт, который тянет `league-of-legends` events и классифицирует рынки (match vs season vs prop).
2. **Слой цен:** привязка `token_id` ↔ вопрос ↔ тип модели (series / map / prop).
3. **MVP модели:** только **match winner** и **series winner** через MC по картам + один источник силы команд (elo из ваших данных или импорт).
4. **Live:** подписка на обновление счёта → пересчёт `p_model` между картами.
5. **Расширение:** тоталы/гандикапы после того, как калибровка по истории стабильна.

---

## 5. Связь с MiroFish

MiroFish можно кормить **снимком**: цены Polymarket + краткий state матча + результат MC (распределение исходов, а не одна точка) + явные допущения. Это помогает **сценарному** анализу, а ставочный размер — отдельно через Kelly/fractional Kelly от `edge` и дисперсии симуляции.

## 6. Дисклеймер

Рынки предсказаний и беттинг регулируются по-разному; модели дают **оценки**, не гарантии. Используйте ответственно.
